# Post View Card 设计方案

本文档记录当前已确认的 `post-view` 卡片视觉方案。当前方案先在
`public/design-demos/post-view-card-demo.html` 中验证，后续再应用到正式组件
`src/components/posts/view/cards/BaseCard.astro`。

## 设计目标

- 卡片要像手机视图一样，以固定比例的竖向 media card 承载文章预览。
- 多张卡片同时展示时，要优先保证扫读效率，避免每张卡片出现过重的信息面板。
- 背景图是第一视觉层，使用全尺寸黑色遮罩提高标题、正文和 tag 的可读性。
- 卡片内容保持三段式：顶部标题，中部正文预览，底部信息。
- Ghost tag 的分类结果只作为卡片四角的信息内容展示，不显示 `post_type` 等字段名。

## 当前固定配置

| 项目             | 方案                                     |
| ---------------- | ---------------------------------------- |
| Structure        | `media`                                  |
| Media Copy       | `centered`                               |
| 标题策略         | `auto`                                   |
| 中文正文预览上限 | `145` 字符                               |
| 日文正文预览上限 | `145` 字符                               |
| 英文正文预览上限 | `265` 字符                               |
| 背景图处理       | 全尺寸图片 + 全尺寸黑色遮罩              |
| Demo URL         | `/design-demos/post-view-card-demo.html` |

## 信息结构

卡片保持三段式内容结构：

1. 顶部标题区
   显示文章标题。标题居中，使用自动密度策略适配短标题、常规标题、长标题和压力测试标题。

2. 中部正文区
   显示文章正文预览。正文居中，宽度受控，避免在卡片墙中形成大块满版文字。

3. 底部信息区
   显示发布日期、阅读时长等辅助信息。当前 demo 仍保留 mock 信息，正式实现应接真实字段。

四角 tag 信息独立于三段式正文结构：

| 位置   | 字段                | 内容来源              | 展示规则                                                                     |
| ------ | ------------------- | --------------------- | ---------------------------------------------------------------------------- |
| 左上角 | `post_type`         | Ghost `type-` tag     | 显示实际类型，如 `article` / `music` / `video` / `gallery`                   |
| 右上角 | `post_series`       | Ghost `series-` tag   | 显示系列名称；无有效系列时隐藏                                               |
| 左下角 | `post_general_tags` | 普通公开 tag          | 从下至上反向纵向堆叠，默认最多显示 3 个；超过 3 个默认隐藏，hover 时展开全部 |
| 右下角 | `post_category`     | Ghost `category-` tag | 显示实际分类；无有效分类时隐藏                                               |

视觉层只显示实际 tag 值，不显示字段名。

## Ghost Tag 映射

Ghost 后端标签由 `src/api/adapters/ghost.ts` 统一分类：

| Ghost tag slug       | 前端字段            | 说明                          |
| -------------------- | ------------------- | ----------------------------- |
| `type-*`             | `post_type`         | 文章媒介类型                  |
| `category-*`         | `post_category`     | 内容分类                      |
| `series-*`           | `post_series`       | 系列，使用 tag name 展示      |
| 非 `hash-*` 普通 tag | `post_general_tags` | 泛用 tag                      |
| `hash-*`             | 不展示              | 内部 tag，例如语言和 i18n key |

正式卡片实现应直接使用这些适配后的字段，不在 UI 组件中重新解析 Ghost 原始 tag。

## 标题规则

标题策略固定为 `auto`。目标不是无限展示完整标题，而是在卡片固定尺寸内保留最强可读性。

当前 demo 的自动密度规则：

| 语言        | Medium 阈值 |   Long 阈值 | Stress 阈值 |
| ----------- | ----------: | ----------: | ----------: |
| 中文 / 日文 | `> 18` 字符 | `> 32` 字符 | `> 50` 字符 |
| 英文        | `> 28` 字符 | `> 58` 字符 | `> 90` 字符 |

对应视觉规则：

| 密度           |   字号 | 最大行数 | 最大宽度 |
| -------------- | -----: | -------: | -------: |
| Short / Medium | `34px` |   `4` 行 |   `11ch` |
| Long           | `30px` |   `5` 行 |   `13ch` |
| Stress         | `27px` |   `5` 行 |   `15ch` |

超出最大行数时允许截断。正式实现需要保留 line clamp，不能让标题挤压正文区或四角 tag。

## 正文预览规则

正文预览按语言设置硬上限：

| 语言     | 最大字符数 |
| -------- | ---------: |
| English  |      `265` |
| Chinese  |      `145` |
| Japanese |      `145` |

正文预览居中显示，目标边界是压力测试内容仍能在 `10` 行以内展示，不破坏卡片结构。

正式实现建议：

- 先按当前 locale 选取字符上限。
- 截断时保留完整字符，不做字节级截断。
- 超过上限后追加省略号。
- 继续保留 CSS line clamp，作为异常内容的第二道保护。

## 视觉规则

- 卡片比例使用竖向手机视图，不做横向文章卡。
- 背景图必须覆盖整个卡片，`object-fit: cover`。
- 黑色遮罩覆盖整个背景图，遮罩强度以正文和标题可读为准。
- 标题和正文居中，避免底部黑色内容面板造成突兀割裂。
- 四角 tag 使用轻量 pill，不增加字段标题。
- 左下角 general tags 从下至上反向纵向堆叠，最多为 3 个 tag 预留默认显示空间；超过 3 个只在 hover 时展示，并继续向上展开。
- 日期固定在底部中线位置，不能参与四角 tag 的 grid 高度计算；左下角 tag 展开时日期位置不能移动。
- 实际 `post-view` 是横向 snap 轨道，同屏可能出现 3-4 张卡，卡片外层 `.post-card-wrapper` 已承担点击目标和轻微 hover scale；正式实现时不要在内层卡片再叠加整卡 scale。
- 卡片动效以局部 hover 反馈为主：内层卡片保持布局稳定，只增强边框和阴影；背景图只做轻微放大，不做上移；遮罩高光渐显，额外 tag 使用约 `280ms` 的柔和 ease-out 淡入展开；不使用会占用结束态 `transform` 的入场动画。
- 四角 tag 不参与点击和滚动，不应改变卡片整体点击目标。
- 多卡片展示时，tag 和正文不能互相遮挡。

## 响应式验收范围

后续把方案应用到正式页面后，需要在以下 viewport 做人工视觉验收：

| 设备/场景    | 建议 viewport            |
| ------------ | ------------------------ |
| Android 手机 | `360x800`、`393x873`     |
| iPhone       | `390x844`、`430x932`     |
| iPad Pro     | `1024x1366`、`1366x1024` |
| 1080P 显示器 | `1920x1080`              |
| 2K 显示器    | `2560x1440`              |
| 4K 显示器    | `3840x2160`              |

每个 viewport 至少检查：

- 卡片比例是否稳定。
- 长标题是否被正确压缩或截断。
- 中英日正文预览是否在边界内。
- 四角 tag 是否可读且不遮挡标题/正文/底部信息。
- 多卡片并排或滑动展示时是否有明显拥挤、裁切或溢出。

- 英文正文压力数据限制为 `265` 字符。

## 后续落地约束

将 demo 方案应用到正式 `BaseCard.astro` 时，需要遵守：

- 使用现有 `Post` 数据结构和 Ghost adapter 输出字段。
- 不在卡片内展示 `post_type` / `post_series` / `post_general_tags` / `post_category` 字段名。
- `default`、空字符串和缺失字段不应展示为可见 tag。
- 生产实现应保留当前 demo 的字符上限和标题 auto 策略。
- 正式页面验收前，需要用真实 Ghost 数据覆盖中英日、短/中/长标题、短/中/长正文。
